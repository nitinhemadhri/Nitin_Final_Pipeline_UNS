3-Phase Pipeline Architecture
Each phase is independent. Phase 1 is ready to run. Phase 2 and 3 are next.
Phase 1

MES DB → PostgreSQL

Poll 819 tables every 60 seconds via ODBC. Store all rows in PostgreSQL raw_data with dual-offset tracking. 8 async workers, 200-row batch inserts, queue.Queue decoupling.

✓ Ready to Run
Phase 2

PostgreSQL → Kafka

kafka_producer.py reads PENDING_KAFKA rows, looks up UNS from table_metadata, builds field-level payload, sends to Kafka topic in UNS format on Linux VM.

→ Next Up
Phase 3

Kafka → S3 Parquet

kafka_consumer.py batches 1000 rows, converts to Parquet with pyarrow + snappy, uploads to S3. ACK triggers offset update. Zero data loss guaranteed.

Scripts
All Python Scripts
config.py
Always Loaded
All DB credentials, IPs, UNS function, settings. Single source of truth for all scripts.
01_setup_db.py
Run Once
Creates all 4 PostgreSQL tables with indexes and partitions. Run before everything else.
02_load_tables.py
Run Once
Reads 3 CSVs → loads 819 tables into offset_tracker and pipeline_monitor.
03_first_load.py
Run Once
Pulls 1 month historical data (from 2026-02-01) from all 819 MES tables. Populates raw_data.
04_pipeline.py
Always Running
Real-time incremental poller. 8 workers, 60s cycle, queue.Queue decoupling, dual-offset logic.
05_check_status.py
Run Anytime
Shows DB connections, row counts, partition sizes, push_status counts, top 10 tables.

# 3. Load 819 tables (once)

# 4. Pull 1 month history (once)
py -3.12 03_first_load.py

# 5. Verify everything
py -3.12 05_check_status.py

# 6. Start live pipeline (keep running)
py -3.12 04_pipeline.py

1
 Offset zero treated as missing if not offset: treats 0 as None if offset is None: ✓
2
 Thread explosion Thread spawned per MQTT publish inside pool queue.Queue decouples — no nested threads ✓
3
 60s sleep on DB fail time.sleep(60) blocks worker thread Return immediately on error, next cycle picks up ✓
4
 No query timeout DB query hangs forever pyodbc.connect(timeout=15) ✓

 Memory explosion No batch limit — millions of rows in memory BATCH_SIZE=200, QUEUE_MAXSIZE=50000 ✓
